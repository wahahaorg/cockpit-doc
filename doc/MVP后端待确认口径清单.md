# MVP 后端待确认口径清单

文档版本：V0.1  
用途：记录不阻塞确定性后端开发、但上线真实数据前需要产品或财务确认的业务口径。

## 1. 当前暂行口径

| 模块 | 待确认事项 | 当前暂行实现 | 影响 |
| --- | --- | --- | --- |
| 30/60/90 天预测 | 现金安全线是否长期为 0 元 | 使用 0 元 | 影响缺口金额和跌破日期 |
| 30/60/90 天预测 | `planned`、`pending` 是否占用预测现金 | 两者与 `approved` 均进入基础预测 | 影响预测余额 |
| 30/60/90 天预测 | 实际付款数据缺失时如何识别已付款 | 依赖计划支出状态及时更新 | 可能重复计算支出 |
| 回款风险 | 临期是否按自然日还是工作日 | 未来 0～2 个自然日为黄色风险 | 影响 Top 3 候选 |
| 回款风险 | 红色风险阈值 | 逾期 30 天，或逾期且未回金额达到 100 万元 | 影响老板拍板事件 |
| 付款建议 | 计划日期早于基准日的已审批支出如何试算 | 视为基准日当天待支付 | 避免逾期支出从预测中消失 |
| 付款建议 | 同日期、同刚性程度是否需要类别优先级 | 按支出编号稳定排序 | 影响逐笔试算顺序 |
| 老板拍板 | 决定后是否自动生成执行任务 | 当前只保存决定，不自动执行 | 不影响拍板留痕 |
| AI 解释 | 是否缓存及缓存失效规则 | 暂不实现 AI 接口 | 不阻塞确定性业务接口 |

## 2. 当前明确不交给 AI 判断的内容

- 现金余额、缺口金额和日期；
- 回款风险等级和 Top 3 排序；
- 付款资格、付款顺序和付款建议；
- 是否升级老板拍板；
- 老板可选处理方案是否合法。

AI 后续只读取结构化证据生成解释，不修改规则计算结果。

## 3. 变更原则

上述口径确认后，应优先调整集中规则参数或共享计算模块，不在控制器、页面接口和前端分别修改，避免同一指标出现多套公式。
