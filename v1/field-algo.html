CEO现金流驾驶舱 V1.0 字段、算法与数据来源清单 ​

文档版本：V1.0
适用项目：CEO现金流经营决策系统 V1.0
核心用途：明确每个指标的业务定义、展示命名、计算规则、数据来源、维护责任人与复核责任人。
确认原则：所有核心字段、算法、阈值必须由 CEO 和 CFO 共同确认。

一、文档目标 ​

本清单是 CEO现金流驾驶舱 V1.0 的核心规则文档。

它优先级高于前端工程化任务书，因为前端页面、后端接口、数据库模型和 AI 回答依据，都必须基于本清单确定。

本清单要解决四个问题：

  1. 页面上每个数字叫什么；
  2. 这个数字的财务定义是什么；
  3. 这个数字怎么计算；
  4. 这个数字从哪里来、谁维护、谁复核。

二、确认边界 ​

本文件分为业务确认项和技术实现项，二者不能混在一起决策。

2.1 CEO + CFO 必须确认 ​

  • CEO 展示名称；
  • CFO 财务定义；
  • 业务含义；
  • 计算公式；
  • 阈值规则；
  • 数据来源；
  • 数据维护人；
  • 数据复核人；
  • 复核状态；
  • 是否可用于正式经营判断。

2.2 技术团队确认 ​

  • 英文字段名；
  • 数据库字段类型；
  • 接口字段结构；
  • 枚举编码；
  • 前后端命名规范；
  • 是否拆表、是否冗余、是否需要索引。

2.3 命名原则 ​

文档中的英文技术字段名均为建议字段名，只用于帮助工程团队理解，不作为 CEO/CFO 的业务确认项。

最终技术字段名应在《接口与数据模型评审清单》中由前端、后端和全栈工程师确认。

三、字段确认原则 ​

  1. CEO 看到的名称必须清晰、直观、便于经营判断；
  2. CFO 确认的财务定义必须准确、可追溯、可复核；
  3. 所有指标必须有数据来源，不能只有页面展示；
  4. 所有计算字段必须有公式和参数；
  5. 未经 CFO 复核的数据，不能作为正式经营结论；
  6. AI 助手回答经营问题时，必须引用本清单中的字段、算法和数据来源。

六、核心算法规则 ​

6.1 本月现金流缺口 ​

展示名称：本月现金流缺口

计算公式：

text
本月现金流缺口 = 本月计划支出 - 本月可用现金 - 本月预计回款
1

判断规则：

  • 如果本月现金流缺口 <= 0，状态为安全；
  • 如果本月现金流缺口 > 0，状态为存在缺口；
  • 如果缺口金额超过 CFO 设置的安全阈值，状态为高风险。

待 CFO 确认：

  • 本月可用现金是否包含短期理财或备用金；
  • 本月预计回款是否只包含高确定性回款；
  • 本月计划支出是否包含待审批支出；
  • 安全阈值金额或比例。

6.2 回款逾期判断 ​

展示名称：回款是否逾期

计算公式：

text
逾期天数 = 当前日期 - 约定到账日期
1

判断规则：

  • 当前日期 <= 约定到账日期：未逾期；
  • 当前日期 > 约定到账日期，且未到账：逾期；
  • 已部分到账时，需要展示未到账金额和剩余逾期风险。

待 CFO 确认：

  • 约定到账日期以合同为准，还是以历史到账日为准；
  • 节假日是否顺延；
  • 部分到账是否仍触发预警；
  • 不同客户是否允许不同宽限期。

6.3 回款风险等级 ​

展示名称：回款风险等级

建议初版规则：

风险等级判断条件
正常未逾期，或逾期 0 天
关注逾期 1-3 天，或金额较小但未到账
高风险逾期 4-7 天，或金额较大
严重风险逾期超过 7 天，或关键客户大额未到账

待 CFO 确认：

  • 金额较大的标准；
  • 关键客户名单；
  • 不同业务线是否有不同风险阈值；
  • 历史到账习惯是否参与风险等级判断。

6.4 支出是否超预算 ​

展示名称：是否超预算

计算公式：

text
支出后预算占用 = 已发生金额 + 已审批待付款金额 + 当前申请金额
是否超预算 = 支出后预算占用 > 对应预算额度
1
2

待 CFO 确认：

  • 预算按月、季度还是年度控制；
  • 预算按部门、项目、科目还是业务线控制；
  • 是否允许负责人特批；
  • 超预算是否直接阻断，还是只预警。

6.5 支出是否影响现金流 ​

展示名称：是否影响现金流

计算公式：

text
付款后可用现金 = 当前可用现金 - 当前申请付款金额
付款后现金流缺口 = 本月计划支出 - 付款后可用现金 - 本月预计回款
1
2

判断规则：

  • 付款后现金流缺口 <= 0：原则可批；
  • 付款后现金流缺口 > 0：需谨慎；
  • 付款后现金低于安全阈值：高风险，建议升级 CFO 或 CEO。

待 CFO 确认：

  • 现金安全阈值；
  • 哪些支出属于刚性支出；
  • 哪些支出可以延后；
  • 哪些支出必须 CEO 确认。

九、AI 回答引用规则 ​

AI 助手回答现金流、回款、支出审批问题时，必须遵守：

  1. 只能基于已确认字段、已复核数据和明确规则回答；
  2. 必须说明结论依据；
  3. 涉及金额、日期、风险等级时，必须说明数据来源；
  4. 数据未复核时，必须提示“该数据仍待 CFO/财务复核”；
  5. 不能编造缺失数据；
  6. 不能自行修改财务口径；
  7. 不能直接给出付款执行指令。

  • [ ] 核心公式确认；
  • [ ] 数据来源确认；
  • [ ] 数据维护人确认；
  • [ ] 数据复核人确认；
  • [ ] 现金安全阈值确认；
  • [ ] 回款逾期阈值确认；
  • [ ] 回款风险等级规则确认；
  • [ ] 支出预算判断规则确认；
  • [ ] AI 回答依据规则确认；
  • [ ] 未复核数据展示口径确认。

十一、工程团队待确认项 ​

以下事项不需要 CEO/CFO 直接确认，但需要在接口与数据模型评审中由技术团队确认：

  • [ ] 最终英文字段名；
  • [ ] 数据库存储类型；
  • [ ] 金额精度和币种处理；
  • [ ] 日期和时区规则；
  • [ ] 枚举值编码；
  • [ ] 接口返回结构；
  • [ ] 计算字段是否落库；
  • [ ] 任务列表是否独立建模；
  • [ ] 用户、客户、业务线之间的关联关系；
  • [ ] 数据复核状态是否作为通用字段。
