## 2. 基础派生值

### 2.1 应收累计回款

```text
累计回款 = Σ 有效实际回款金额（按应收编号关联）
```

### 2.2 应收未回金额

```text
未回金额 = max(应收金额 - 累计回款, 0)
```

### 2.3 逾期天数

```text
如果未回金额 = 0：逾期天数 = 0
如果统计日期 <= 约定到账日期：逾期天数 = 0
否则：逾期天数 = 统计日期 - 约定到账日期
```

部分回款后仍有余额时，按剩余未回金额继续计算逾期，待 CFO 确认。

## 3. 驾驶舱指标

### 3.1 当前可用现金

V3 默认取每个账户不晚于统计日期的最新一条余额快照：

```text
当前可用现金 = Σ 各账户最新可用余额
```

受限金额是否需要再扣减，取决于财务提供的“可用余额”定义。

### 3.2 本月预计回款

```text
有效预计日期 = 若预计到账日期存在则取预计到账日期，否则取约定到账日期
本月预计回款 = Σ 有效预计日期落在本月且未取消的应收未回金额
```

争议点：已逾期但仍未到账的应收是否继续计入本月预计回款。

### 3.3 本月实际回款

```text
本月实际回款 = Σ 回款日期落在本月的有效回款金额
```

### 3.4 本月计划支出

```text
本月计划支出 = Σ 计划日期落在本月、状态属于 planned/pending/approved 的计划金额
```

是否排除可延后支出、是否只统计已审批支出，待 CFO 确认。

### 3.5 本月现金流缺口

```text
期末预测现金 = 当前可用现金 + 本月剩余预计回款 - 本月剩余计划支出
本月现金流缺口 = max(-期末预测现金, 0)
```

“剩余”默认指统计日期至月末，避免将本月已发生回款和支出重复计算。若只能获得整月计划，则需明确计划金额中是否包含已发生部分。

### 3.6 逾期回款金额

```text
逾期回款金额 = Σ（约定到账日 < 统计日期且未回金额 > 0）的未回金额
```

### 3.7 今日待处理事项

```text
今日待处理 = 未关闭且 due_date <= 统计日期的任务
```

任务由确定性规则生成，不由大模型自行创建经营事实。

## 4. 四张决策卡规则

| 卡片 | V3 判断 | 依赖数据 | 待确认 |
| --- | --- | --- | --- |
| 钱够不够 | 缺口 > 0 为红；期末预测现金低于安全阈值为黄；否则绿 | 余额、预计回款、计划支出 | 安全阈值、预计回款可信折扣 |
| 钱该进没进 | 存在逾期未回为黄；达到红灯天数或大额阈值为红 | 应收、实际回款 | 风险天数和金额阈值 |
| 钱能不能花 | 拟支出后缺口 > 0 为红；低于安全阈值为黄；否则绿 | 可用现金、预计回款、计划支出、拟支出 | 是否区分刚性支出及预算 |
| 谁在处理 | 风险事项有负责人且有未关闭任务；无负责人或超期未处理升级 | 责任人、任务、跟进记录 | 升级时限、老板介入条件 |

## 5. 回款风险等级 V0.1

按最高风险命中：

```text
退出候选：未回金额 = 0
绿：未回金额 > 0，且距离到期超过 2 天
黄：距离到期 0-2 天，或逾期 1-29 天且未触发红灯金额条件
红：逾期 >= 30 天，或逾期且未回金额 >= 1,000,000 元
```

风险规则只作真实样本验证，不作为正式经营结论。

## 6. 任务生成 V0.1

| 触发条件 | 任务类型 | 默认责任人 | 老板介入 |
| --- | --- | --- | --- |
| 本月现金流缺口 > 0 | cash_gap | 财务负责人/CFO | 不单独生成拍板事件，作为回款/付款事件证据 |
| 应收逾期且未回金额 > 0 | collection_overdue | 应收责任人 | 红灯时是 |
| 计划支出导致现金缺口 | expense_review | 支出责任人/财务 | 已审批刚性支出导致缺口时是 |
| 应收或支出缺少责任人 | owner_missing | 数据管理员 | 超过 1 天未补全时是 |

同一来源、任务类型和处理日不得重复生成未关闭任务。

## 7. 计算结果必须返回的审计信息

每个指标接口除数值外还应返回：

- `asOfDate`：计算基准日；
- `periodStart` / `periodEnd`：统计范围；
- `batchId`：数据批次；
- `ruleVersion`：规则版本；
- `reviewStatus`：待 CFO 复核/已复核；
- `sourceCount`：参与计算的记录数；
- `formulaCode`：稳定公式编码；
- `warnings`：数据缺失或口径提示。
