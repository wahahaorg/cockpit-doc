# 回款风险 Top 3 模块 MVP 技术方案初稿

文档版本：V0.1  
状态：讨论稿，风险阈值待产品与财务确认  
目标读者：产品、财务、后端开发和测试人员

## 1. 模块目标

本模块识别当前最需要关注的应收款，回答以下问题：

- 哪些应收尚未全部到账；
- 哪些应收已经逾期或即将到期；
- 当前未回金额是多少；
- 应优先关注哪三笔应收；
- 每笔风险应收由谁负责。

风险等级和排序由确定性规则计算。AI 不参与金额、到期状态、风险等级或 Top 3 排序，只能解释已经计算出的结果。

## 2. MVP 范围

### 2.1 本期实现

- 计算每笔应收的累计实际回款和未回金额；
- 计算逾期、今日到期、临期和正常状态；
- 按统一规则生成红、黄、绿风险等级；
- 从红色和黄色候选中选出回款风险 Top 3；
- 返回客户、金额、负责人、到期状态和最近跟进信息；
- 支持从结果追溯到应收款和实际回款记录。

### 2.2 本期不实现

- 使用 AI 猜测客户是否会付款；
- 根据客户历史行为训练违约概率模型；
- 节假日或工作日历计算；
- 客户信用评分和外部征信；
- 自动发送催收消息或执行资金操作。

## 3. 数据来源

### 3.1 主表：应收款

回款风险以应收款为主表。主要使用：

- 应收编号；
- 客户；
- 应收金额；
- 约定到账日期；
- 当前预计到账日期；
- 责任人；
- 业务状态。

### 3.2 关联表：实际回款

实际回款用于扣除已经到账的金额。一笔应收可以关联多笔实际回款。

```text
一笔应收款 1 → N 笔实际回款
```

### 3.3 辅助数据：回款跟进记录

如果需要展示“今日待反馈”“跟进中”或最近跟进内容，可关联回款跟进记录，读取最近一次跟进时间、内容和下次跟进日期。

跟进记录只影响页面状态和任务处理，不直接改变应收金额、未回金额和风险等级。

## 4. 计算基准

每次计算必须指定统计日期 `asOfDate`，本文记为 `D`。

所有应收和实际回款必须来自同一个已发布数据批次。未发布、已作废或校验失败的数据不得进入正式计算。

## 5. 未回金额计算

对每笔有效应收：

```text
累计实际回款
= Σ 关联到该应收的有效实际回款金额

未回金额
= max(应收金额 - 累计实际回款, 0)
```

处理规则：

- 未回金额等于 0，表示已经全部回款，不进入风险候选；
- 部分回款后仍有余额，继续按照剩余未回金额判断风险；
- 累计回款超过应收金额时，未回金额按 0 处理，并返回数据异常警告；
- 业务状态为 `cancelled` 或数据状态无效的应收不参与计算。

## 6. 到期状态计算

逾期和临期以约定到账日期 `agreed_due_date` 为判断依据，不使用预计到账日期替代合同约定。

```text
距到期天数 = 约定到账日期 - D
```

MVP 使用自然日：

| 条件 | 到期状态 | 页面建议文案 |
| --- | --- | --- |
| 距到期天数 < 0 | 已逾期 | 逾期 N 天 |
| 距到期天数 = 0 | 今日到期 | 今日到期 |
| 距到期天数 = 1 | 临期 | 明日到期 |
| 距到期天数 = 2 | 临期 | 还有 2 天到期 |
| 距到期天数 > 2 | 正常 | 距到期 N 天 |

逾期天数定义：

```text
如果未回金额 = 0：逾期天数 = 0
否则：逾期天数 = max(D - 约定到账日期, 0)
```

## 7. 风险等级

MVP 按最高风险条件命中：

### 7.1 红色风险

满足任一条件：

```text
逾期天数 >= 30 天
或
已经逾期且未回金额 >= 1,000,000 元
```

### 7.2 黄色风险

未触发红色风险，并满足任一条件：

```text
逾期天数为 1～29 天
或
距到期天数为 0～2 天
```

黄色风险因此同时包含轻度逾期、今日到期和未来两天内临期的应收。

### 7.3 绿色状态

```text
未回金额 > 0
且
距到期天数 > 2 天
```

绿色代表当前未临期，不代表应收已经到账。已经全部回款的应收直接退出当前风险候选，不使用绿色风险继续展示。

## 8. Top 3 候选和排序

### 8.1 候选范围

Top 3 只从红色和黄色风险中选择：

```text
候选应收
= 有效应收
且未回金额 > 0
且风险等级属于 red 或 yellow
```

如果候选不足三笔，则按实际数量展示，不使用模拟数据补足。

### 8.2 排序规则

排序键依次为：

```text
1. 风险等级：红色优先于黄色
2. 到期紧急程度：逾期 > 今日到期 > 明日到期 > 还有 2 天到期
3. 同类逾期：逾期天数更多的优先
4. 条件仍相同：未回金额更大的优先
5. 条件仍相同：应收编号升序，保证结果稳定
```

取排序后的前三笔作为页面 Top 3。

## 9. 页面展示字段

每条 Top 3 记录至少展示：

- 客户名称；
- 应收编号；
- 原始应收金额；
- 已回金额；
- 未回金额；
- 约定到账日期；
- 逾期或临期文案；
- 风险等级；
- 责任人；
- 最近跟进时间和状态（存在跟进数据时）。

页面中的金额、日期、负责人和风险等级必须可以追溯到参与计算的业务记录和规则版本。

## 10. 计算示例

假设统计日期为 7 月 5 日：

| 应收 | 应收金额 | 已回金额 | 未回金额 | 约定到账日 | 结果 |
| --- | ---: | ---: | ---: | --- | --- |
| A | 300 万 | 0 | 300 万 | 7 月 4 日 | 逾期 1 天，因逾期且金额超过 100 万，红色 |
| B | 200 万 | 50 万 | 150 万 | 7 月 6 日 | 明日到期，黄色 |
| C | 160 万 | 0 | 160 万 | 7 月 7 日 | 还有 2 天到期，黄色 |
| D | 80 万 | 80 万 | 0 | 6 月 30 日 | 已全部回款，不进入候选 |
| E | 90 万 | 0 | 90 万 | 7 月 20 日 | 未临期，绿色，不进入 Top 3 |

最终 Top 3 为 A、B、C。

## 11. 输出结果最低要求

结果至少包含：

- 统计日期、数据批次和规则版本；
- 风险候选总数及红色、黄色数量；
- 排序后的 Top 3；
- 每笔应收的原始金额、累计回款、未回金额和到期状态；
- 风险等级及命中的规则；
- 参与计算的应收和实际回款来源标识；
- 超额回款、日期缺失或责任人缺失等数据警告。

金额必须使用十进制定点计算，禁止使用二进制浮点数直接计算财务金额。

## 12. MVP 验收标准

- 已全部回款的应收不进入风险候选；
- 部分回款按照剩余未回金额判断风险；
- 今日到期及未来两天内到期的应收进入黄色候选；
- 红色始终排在黄色之前；
- 相同输入、统计日期和规则版本产生完全一致的 Top 3；
- 候选不足三笔时不使用模拟数据补足；
- 每个结果可以追溯到应收款和实际回款记录。

## 13. 待确认口径

1. 临期长期采用 2 个自然日，还是改为工作日；
2. 红色大额阈值是否采用 100 万元；
3. 红色逾期阈值是否采用 30 天；
4. 客户等级或业务线是否参与风险排序；
5. 已承诺新到账日期是否只展示，还是影响风险等级；
6. 最近跟进超时是否影响回款风险，还是只影响风险任务等级。
